# Changelog

All notable changes to the TypeScript Best Practices skill will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.1] - 2026-01-16

### Changed
- Renamed directory from `typescript/` to `typescript-best-practices/` to match skill name
- Enhanced description with file type patterns (.ts, .tsx, .d.ts, tsconfig.json)
- Added slash command invocation path (/typescript-best-practices)
- Added TypeScript error patterns for automatic activation (TS2322, TS2339)
- Added synonym coverage (TS, tsc, type-checking)
- Added negative cases clarifying what the skill does NOT cover

## [0.1.0] - 2026-01-15

### Added
- Initial release with 42 rules across 8 categories
- Type System Performance rules (7 rules)
- Compiler Configuration rules (6 rules)
- Async Patterns rules (5 rules)
- Module Organization rules (5 rules)
- Type Safety Patterns rules (6 rules)
- Memory Management rules (5 rules)
- Runtime Optimization rules (6 rules)
- Advanced Patterns rules (3 rules)
- Progressive disclosure structure (SKILL.md, AGENTS.md, individual rule files)
